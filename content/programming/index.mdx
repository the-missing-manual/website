Most advice about learning to program is either too abstract or too specific. "Just build things" tells you nothing when you don't know what to build. "Learn X first" answers the wrong question.

This section covers the technical skills: how to approach algorithms, what to build, how to write code that other people can read, and how to choose the tools you work with.

<Mark>The right question isn't "what language should I learn." It's "what do I want to build, and what do I need to understand to build it."</Mark>

That reframe matters because it changes what you do next. A language is a tool. A project is a reason. Without the reason, the tool sits in a drawer.

This guide does not recommend specific courses, platforms, or tools. It does not rank languages or say one approach is better than another. The goal is to give you the right questions to ask so you can find answers that fit your situation.

### Common Questions

<Accordions>
  <Accordion title="Which programming language should I learn first?">
    The wrong question. Pick a project you want to build, then use the language that project needs. A finished project teaches more than months of comparing languages. See [Choosing a Language](/programming/language-choice).
  </Accordion>
  <Accordion title="Should I focus on DSA or on building projects?">
    Both, but they do different jobs: DSA gets you past the filter, projects get you the offer. If you have to sequence them, build a foundation first, then layer DSA on top. See [DSA](/programming/dsa) and [Dev](/programming/dev).
  </Accordion>
  <Accordion title="How many DSA problems do I need to solve?">
    Volume is the wrong target. Fifty patterns you can recognize beat a thousand problems you memorized. See [DSA](/programming/dsa).
  </Accordion>
  <Accordion title="I have projects but weak DSA. Am I in trouble?">
    No — your build experience is a multiplier once you clear the Online Assessment, which is almost pure DSA. Close the DSA gap and lean on projects in the live rounds. See [Technical Interviews](/programming/interviews).
  </Accordion>
</Accordions>
