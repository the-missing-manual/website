---
title: Awesome Lists
description: What curated GitHub lists are, how to find them, and how to use them without drowning.
---

GitHub has a large collection of community-maintained resource lists called "awesome" lists. Each one curates resources around a specific topic: a domain, a tool, a concept, a programming area. Knowing they exist and knowing how to find the right one is useful.

### What They Are

An awesome list is a GitHub repository. Someone decided to collect links on a topic, organized them into categories, and opened it to contributions. The most popular lists are maintained by many contributors and updated regularly. The quality varies, but patterns emerge quickly once you've read a few.

The original concept, and a list of awesome lists, lives at [github.com/sindresorhus/awesome](https://github.com/sindresorhus/awesome). Search that repository to find lists on almost any technical topic.

### How to Find Them

On GitHub: search `awesome [topic]` in the repository search. Filter by Stars (high to low) to find the most maintained ones.

On a general search engine: `awesome [topic] github`

Examples of the kinds of lists that exist:
- awesome algorithms
- awesome system design
- awesome open source
- awesome [any domain you can think of]

### How to Use Them

The mistake is reading a list top to bottom and adding every link to a bookmarks folder you'll never open again.

A better approach: open the list with a specific question. "I need to understand X." "I'm looking for a resource that covers Y in depth." Scan the relevant section. Pick one thing to read or work through. Close the list.

Return when you have another specific question.

<Mark>An awesome list is a starting point, not a curriculum. Don't try to use all of it.</Mark>

### Evaluating What You Find

A link being on an awesome list means someone added it. It doesn't mean it's accurate, current, or worth your time.

Before spending significant time on a resource, check:

- When was it last updated?
- Does it link to primary sources or does it mostly summarize others?
- Do other credible sources reference it?
- Is the content accurate for the areas you already know?

The fastest check: read a section on something you already understand well. If it's correct and clear on that, it's probably trustworthy on the things you don't know yet.
