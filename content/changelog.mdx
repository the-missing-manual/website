## v1.4.0 — July 10, 2026

A documentation-quality pass applying the principles from Fuma Nama's [How to Write Good Documentation](https://www.fuma-nama.dev/blog/good-docs) across the whole guide.

### Added

- **Page feedback widget** — every page now ends with a "Was this page helpful?" prompt. Yes/No responses are sent to PostHog, alongside one-click **Suggest an edit** and **Report a problem** links that open the source file or a pre-filled GitHub issue (these links work with no analytics at all). This closes the feedback loop the guide previously had no mechanism for.
- **FAQ sections** — the Choosing a College, Learning to Program, and Fourth Year pages now answer the questions that come up most often, each cross-linking to the page with the full answer.
- **Deep dive: CTC inflation** — the Placements & ROI page now explains *why* reported CTC runs 20–40% higher than take-home, with a table breaking down variable pay, bonuses, provident fund, and stock.

### Changed

- **Acronyms spelled out on first mention** — CSE, NAAC, NIRF, NBA, GSoC, CTC, LPA, ROI, IoT, CTF, DSA, OS, DBMS, OOP, HR, GATE, MS, MTech, SOP, GRE, and LOR are now expanded the first time they appear, so a reader landing on any page cold is not left guessing.
- **Links Worth Bookmarking** is now a table pairing each resource with what to use it for, instead of a run-on list.
- **Interview process** steps became headings with anchors rather than a numbered list, matching the guidance that meaningful headings beat sequential numbers.
- **Bare URLs became real hyperlinks** in the Resources section (Stack Overflow, the "Smart Questions" guide, the awesome-lists index).

### Fixed

- **Analytics were silently dead.** The PostHog client read `VITE_`-prefixed environment variables, but Waku only exposes `WAKU_PUBLIC_`-prefixed vars to the browser — so page views (and the new feedback events) were never sent, in dev or production. Switched to `WAKU_PUBLIC_POSTHOG_KEY` / `WAKU_PUBLIC_POSTHOG_HOST`, added a `.env.example`, and made the provider skip initialization when unconfigured instead of calling `init(undefined)` and dropping every event.
- The feedback widget now resets per page (keyed by route) instead of carrying its answered state across client-side navigation.
- Replaced mangled em-dashes that rendered as `" , "` across the Choosing a College section (accreditation, placements, sub-branches, admissions, links, and culture pages).
- Configured the repository so **Edit this page** and view-source links resolve correctly instead of silently doing nothing.

## v1.3.0 — June 14, 2026

### Reorganized

- **Navigation order** — Changelog moved to the last position in the sidebar. The reading flow is now: Welcome → Choosing a College → The College Journey → Learning to Program → Resources → Wellbeing → Changelog.
- **Learning to Program section reordered** — Pages now follow a logical progression: Choosing a Language → Dev → Clean Code → DSA → Technical Interviews. Previously started with DSA and Interviews before covering the foundational topics they depend on.
- **Choosing a College index** — Added a card linking to the Links Worth Bookmarking page, which existed in the sidebar but was not featured in the section's index.

### Fixed

- Removed a broken link in the After College page that pointed to a `/careers` section which does not exist.
- Removed duplicate "Core CS Subjects" content from the DSA page. That material now lives only in Technical Interviews, where it belongs. The DSA page cross-references it instead.

## v1.1.0 — June 12, 2026

### Added

Five sections launched with full content:

- **Choosing a College** — 10 pages covering how to build a shortlist, evaluate intake size, assess coding culture, understand internship policies, read sub-branch naming (AI/Data Science/Cyber Security), interpret placement data honestly, navigate admission paths, and factor in location.
- **The College Journey** — Year-by-year breakdown: what to focus on before you arrive, in each of the four years, and immediately after you graduate.
- **Learning to Program** — Where to start if you have no prior experience, how to approach DSA, what "working on projects" actually means, clean code fundamentals, and how to choose a programming language.
- **Resources** — How to search effectively (not just Google things), how to use awesome lists, and how to ask questions in a way that gets useful answers.
- **Wellbeing** — Mental and physical health during placement season and through the degree in general.
